## 1. システム概要

本システムは Raspberry Pi 上で動作する
ハウス（ビニールハウス）開閉制御装置である。

Home Assistant を UI とし、 ローカル Python HTTP サーバ経由で

-   モータ制御
-   環境センサ取得

を行う。

外部公開は Cloudflare Tunnel 経由。

## 3. 動作環境

-   Raspberry Pi OS (Trixie) ※2025年以降の標準リリース
-   Docker (Home Assistant)
-   network_mode: host
-   systemd 管理サービス
-   journald ログ最小化
-   ROM化済み（root ext4 を `ro` マウント、Docker・HA・cloudflared データは USB へ分離）

### GPIO / PWM 方式について

#### なぜ pigpio を使わないのか

Trixie（Pi5）環境では `pigpio` が**実質使用不可**であり、複数の問題が重なっている：

| 問題 | 技術的背景 |
|------|-----------|
| **Pi 5 でハードコード除外** | Pi5 は BCM2712 + RP1 サウスブリッジ構成。GPIO を管理するのは RP1 で、pigpio が依存する `/dev/mem` 経由の DMA アクセスが機能しない。pigpio はリビジョンコード未登録として `"unknown rev code"` で即時終了する |
| **Trixie でビルド不可** | pigpio の `setup.py` は Python の `distutils` モジュールに依存。Python 3.12（Trixie 標準）で `distutils` が標準ライブラリから削除され、Trixie には代替の `python3-distutils` パッケージも存在しない |
| **メンテナンス事実上終了** | 開発者（joan2937 氏）自身が Pi5 対応を「互換性を壊さずには不可能かもしれない巨大なタスク」と GitHub Issue で発言。Pi5 サポートの公式ロードマップなし |

#### なぜ RPi.GPIO を方向制御に使うのか

`RPi.GPIO` も Pi 5 では公式非対応（Raspberry Pi 公式が明言）だが：

-   本システムでは **方向制御（GPIO23 の HIGH/LOW 出力）のみ**に使用
-   Pi 4 系では引き続き動作する
-   **Pi 5 で使う場合は `rpi-lgpio`**（RPi.GPIO 互換の drop-in 代替）を `pip install rpi-lgpio` で追加インストールするとコード変更なしで動作する

#### GPIO ライブラリ全体像（2026年時点）

| ライブラリ | Pi4 | Pi5 | Trixie | PWM | 状況 |
|-----------|:---:|:---:|:------:|-----|------|
| `pigpio` | ○ | **×** | **×**（ビルド不可） | ハードウェア | 実質廃止 |
| `RPi.GPIO` | ○ | △（非公式） | △ | ソフトのみ | 非推奨移行中 |
| `rpi-lgpio` | ○ | ○ | ○ | ソフトのみ | RPi.GPIO drop-in 代替 |
| `lgpio` | ○ | ○ | ○ | ソフトのみ | Raspberry Pi 公式推奨 |
| `gpiozero` | ○ | ○ | ○ | ソフトのみ | 公式推奨（lgpio バックエンド） |
| `python-gpiod` | ○ | ○ | ○ | なし | Linux 標準（GPIO のみ） |
| **sysfs PWM** | **○** | **○** | **○** | **ハードウェア** | **精度 PWM に最善** |

→ モータ制御のように精度が必要な PWM には **sysfs ハードウェア PWM が最も安定**。
`dtoverlay=pwm` を `/boot/firmware/config.txt` に追記して有効化する。

## 5. センサ取得仕様

### 使用センサ

-   SEN0501（温湿度）I2C 0x22
-   SCD41（CO2 / 温湿度）I2C 0x62

### ポート

http://127.0.0.1:8124/metrics

### JSON形式

{ "timestamp": 1700000000, "sen0501": { "temperature_c": 20.1,
"humidity_rh": 40.2 }, "scd41": { "co2_ppm": 900, "temperature_c": 19.9,
"humidity_rh": 41.0 } }

## 6. モータ制御仕様

### GPIO割当

-   DIR: GPIO23
-   PWM: GPIO18 (pwmchip0 / CH0)
-   周波数: 20kHz
-   最大動作時間: 900秒

### API仕様

開閉: GET /run?dir=0\|1&sec=秒数&duty=0-100

停止: GET /stop

ヘルスチェック: GET /health

## 7. フェイルセーフ設計

### 過走防止の二重構成

| レイヤ | 機構 | 説明 |
|--------|------|------|
| **ハードウェア** | モータ内蔵機械式リミットスイッチ | 設定回転数に達すると内部ギヤがスイッチに接触し、内部回路を遮断・自動停止。赤/緑ダイヤルで開閉幅（回転数）を調節 |
| **ソフトウェア** | 最大動作時間 900秒制限 | `motor_run()` の `MAX_SECONDS` で上限を強制。ハードが万一機能しなくても時間でカット |

### その他フェイルセーフ

-   lockファイルによる二重起動防止（`/run/house-motor.lock`）
-   `finally` で必ず `motor_stop` 実行（例外・タイムアウト問わず停止）
-   GPIO cleanup 実施
